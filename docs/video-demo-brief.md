# Bahus Assistant — Презентация демо-стенда

## 1. Что это и зачем

**Bahus Assistant** — B2B-инструмент для менеджеров, работающих с поставщиками алкоголя.
Закрывает три задачи:

- **Загрузка прайсов** — принимает PDF и Excel от поставщиков, разбирает их через ИИ в структурированную таблицу позиций
- **Ведение номенклатуры** — единая база позиций по всем поставщикам с поиском, фильтрами и нормализацией
- **Формирование КП** — собирает коммерческое предложение под запрос клиента, считает маржу, выгружает в Excel

---

## 2. Текущий демо-стенд

**Стек:** Vercel (фронтенд) + Railway (Python API) + Supabase PostgreSQL + n8n + OpenAI

**Что работает прямо сейчас:**
- Загрузка 4 типов документов: Прайс-лист, Нетто-прайс, Промо, Под запрос
- ИИ-разбор PDF через OpenAI Assistants API (GPT-4o + file_search)
- Трёхфазный интерфейс загрузки: форма → отправка → обработка ИИ
- Создание КП: клиент, тема, состав позиций, расчёт маржи
- Экспорт КП в Excel
- Поиск и фильтрация по всей номенклатуре
- Настраиваемые колонки таблицы (drag-to-reorder)

**Статус:** прототип, данные реальные, UI готов к демонстрации.

## 3. Стоимость инфраструктуры

### Серверная инфраструктура (в месяц)

| Компонент | Тариф | Стоимость |
|---|---|---|
| Vercel (фронтенд) | Hobby (free) → Pro при росте | $0–$20/мес |
| Railway (Python API + хранение файлов) | Hobby + usage | ~$15–25/мес |
| Supabase (PostgreSQL) | Free (до 500 МБ) → Pro | $0–$25/мес |
| n8n Cloud (ИИ-пайплайн) | Starter (5 000 запусков) | ~$26/мес |
| **Итого базовая инфраструктура** | | **~$40–100/мес** |

> На старте (до 200 документов/мес) реально уложиться в $40–60/мес на всё.

### Стоимость разбора документов (OpenAI GPT-4o)

Каждый загруженный документ проходит обработку через OpenAI:

| Сценарий | Документов/мес | Стоимость OpenAI |
|---|---|---|
| Малый объём | 50–100 | $3–7/мес |
| Средний объём | 300–500 | $15–30/мес |
| Высокий объём | 1 000+ | $50–80/мес |

**Расчёт:** типовой прайс-лист (10–20 страниц) ≈ $0.04–0.08 за разбор.
Цена зависит от размера и плотности таблиц в документе.

> **Вывод:** при 300 документах в месяц полная стоимость эксплуатации — порядка **$60–90/мес**.

## 4. Что нужно доработать (приоритетный список)

### Высокий приоритет — нужно до продакшна

1. **Авторизация пользователей** — сейчас система без логина. Нужен вход по паролю или корпоративному SSO.
2. **Надёжное хранилище файлов** — загруженные прайсы сейчас на диске Railway (эфемерный). При рестарте контейнера файлы теряются. Нужно переехать на Supabase Storage или S3.
3. **Разбор Excel-файлов** — ИИ сейчас умеет разбирать только PDF. Excel (xls, xlsx) нужно либо конвертировать перед отправкой, либо добавить отдельный парсер.

### Средний приоритет — важно для работы

4. **Минимальные партии и кратность** — многие поставщики указывают в прайсе минимум к заказу (6 бутылок, 12 бутылок, 1 ящик). Это поле нужно извлекать и хранить отдельно, иначе при сборке КП менеджер не видит ограничения.
5. **Срок действия прайса** — дата актуальности документа. Нужна автоматическая пометка устаревших прайсов.
6. **Мультипользовательность** — несколько менеджеров с разграничением доступа по клиентам и поставщикам.
7. **Уведомления** — email/Telegram при завершении разбора документа.

### Нюансы прайс-листов, которые надо учесть

- **Минимальная покупка**: одни поставщики ставят минимум 6 бут., другие 12, третьи — целый ящик (6/12/24). Это влияет на КП — нельзя поставить qty=1, если минимум 6.
- **Кратность**: заказ кратный 6 или 12. Если клиент хочет 10 бутылок — нужно округлять до 12.
- **Условия НДС**: прайс с НДС или без — влияет на расчёт маржи. Сейчас флаг есть на уровне поставщика, но не на уровне отдельной строки.
- **Промо-условия**: часть прайсов содержит временные акции с датой окончания — без этого поля менеджер не знает, актуальна ли цена.
- **Валюта строки**: некоторые поставщики указывают цены в USD или EUR с пересчётом по курсу ЦБ — нужен отдельный курсовой механизм.

## 5. Что показываем в видео (сценарий)

1. **Открываем "Импорт файлов"** → кнопка "Загрузить" → модалка с формой
2. **Загружаем 4 документа по очереди**, меняя тип:
   - `Прайс-лист` — общий прайс поставщика
   - `Нетто-прайс` — закупочные цены для расчёта маржи
   - `Промо` — акционные позиции с ограниченным сроком
   - `Под запрос` — бланк под конкретного клиента
3. **Наблюдаем обработку** — 3 фазы: отправка → ИИ разбирает → результат (N позиций)
4. **Переходим в "КП"** → "Создать новое КП" → выбираем клиента, вводим тему запроса
5. **Показываем сформированное КП** — состав позиций, маржа, итог по сумме
6. **Экспортируем в Excel** — файл готов к отправке клиенту

---

## 6. Ключевой месседж для клиента

> Менеджер тратит 15–30 минут на ручной ввод одного прайса. Bahus делает это за 2–3 минуты включая ИИ-разбор. При 50 прайсах в месяц — это освобождение 12–25 часов работы менеджера ежемесячно.
