# Технический Долг и Бэклог (Tech Debt & Backlog)

Здесь мы фиксируем все идеи, отложенные задачи и технический долг проекта Bakhus.

## 1. Прямой парсинг каталога вин (Скрапинг фото и карточек)
Отложено до момента, когда потребуется более точечный сбор данных, помимо Google Images.
Источники для потенциального парсинга:
* **Wine-Shopper** (`https://wine-shopper.ru/`) — Легкая интеграция. Сервер слабо защищен, можно парсить прямо из бэкенда через `BeautifulSoup` (передавая поисковой запрос через URL).
* **WineStyle** (`https://winestyle.ru/`) — Строгая защита от парсинга (Cloudflare / DDoS Guard). Стандартные запросы блокируются по таймауту. Потребуется поднятие Headless-браузера (Playwright с stealth плагинами) или коммерческий Scraping Proxy (например, ZenRows / ScraperAPI).

**Стратегия:** Каскадный поиск (1. Быстрый Wine-Shopper -> 2. Google Images -> 3. Тяжелый WineStyle).

## 2. Разбор "Адских" Excel-прайсов (с картинками и сложной версткой)
Обработка неструктурированных файлов `.xls` и `.xlsx`, которые выходят за рамки классического табличного формата (pandas).
**Проблемы к решению:**
* **Вшитая графика:** Извлечение картинок (фото бутылок), которые в коде Excel лежат отдельными XML-файлами и привязаны к координатам ячеек, а не к самим строкам. Потребуется кастомный скрипт на `openpyxl` для распаковки `.xlsx` архива, чтения `xl/drawings/drawing.xml` и мэтчинга картинок к рядам.
* **Сложная верстка:** Обработка объединенных ячеек. Придется использовать алгоритмы заливки (forward-fill, `ffill` в pandas), чтобы раскатать сложные заголовки по пустым полям.
* **Альтернативное решение (Vision):** Подход без кодинга сложного Excel-обработчика — программно конвертировать "кривой" Excel в многостраничный PDF-файл (или сделать скриншоты) и скормить его готовому `gpt-4o` пайплайну (Vision), чтобы нейросеть сама связала строки с картинками.
