# Bakhus Assistant

`Bakhus Assistant` — B2B-веб-сервис для работы с прайсами поставщиков: импорт файлов, проверка строк, сборка коммерческих предложений и дальнейшая автоматизация через `n8n`.

## Текущий Статус: Production Ready

Проект перешел в стадию самостоятельного продакшена. Мы отказались от временного Cloud Run мока в пользу полноценного боевого сервера в [Yandex Cloud](http://111.88.144.93).

### Что работает:
- **Полный UI без заглушек**: импорт, разбор позиций, КП и настройки. Весь старый моковый код (вроде `demo-data.js`) выпилен.
- **Институциональное состояние**: начальное состояние (initial state) полностью конструируется из базы данных.
- **Инфраструктура**: Nginx + Python Backend + PostgreSQL база данных, упакованные через Docker Compose.
- **CI/CD**: Полностью автоматизированный деплой через **GitHub Actions**. Любой пуш в `main` обновляет сервер.
- **CORS Прокси**: n8n вебхуки больше не ломаются браузером, они прокидываются безопасно через наш Python API.
- **Фоновая очистка**: Загруженные файлы старше 7 дней автоматически вычищаются воркер-тредом.

## Архитектура по слоям

### 1. Frontend
Легковесный клиент, управляемый состоянием без сборщиков.
- Входная точка: `index.html` → `src/app.js`.
- Динамический роутинг: API-вызовы адаптивно направляются на тот же домен `window.location.origin/api`.

### 2. Backend (Primary)
- **`scripts/postgres_api.py`** — Полноценный API сервер, взаимодействующий с PostgreSQL.
- Решает проблемы с надежностью: автопарсинг файлов `requests`, CORS-проксирование, Background threads.
- Подготовлен Dockerfile: `docker/postgres-api.Dockerfile`.
- Внимание: `scripts/mock_api.py` переведен в разряд *Legacy*, он не должен использоваться для реальной работы.

### 3. Database (PostgreSQL)
Схемы и таблицы, покрывающие все B2B сущности, находятся в `db/init/`. База монтируется в Docker Volume.

### 4. Интеграция с n8n
Асинхронный флоу полностью настроен.
1. Пользователь грузит файл.
2. Бекенд сохраняет его в дисковый кэш.
3. Бекенд толкает n8n-вебхук (через `requests`), перекидывая файл.
4. n8n парсит PDF через OpenAI и вызывает наш `POST /api/webhooks/n8n/import-result`.
5. Бекенд обновляет базу данных.

## Локальная разработка

Для запуска локально *с базой данных*:
```bash
docker-compose up -d
```

*(Обратите внимание: встроенный in-memory mock-режим и скрипты `make up-local` считаются устаревшими и отключены, так как фронтенд требует боевого API-ответа для загрузки).*
